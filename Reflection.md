What accessibility challenges did you face, and how did you address them?
One accessibility challenge I faced was ensuring proper color contrast on the website. To address this, I reviewed accessibility guidelines and used online tools to identify color combinations that provide sufficient contrast. This helped improve the readability of the content while also enhancing the overall visual styling of the page.

How did you ensure that your design was responsive and accessible to all users?

Using CSS media queries helps make the website responsive by adjusting the layout for different screen sizes such as mobile, tablets, and desktops. This ensures that the content displays properly on all devices and supports better usability.

To ensure the design is accessible to all users, semantic HTML elements such as header,main,section are used to create a clear structure. ARIA attributes are added to improve accessibility for screen readers, and alt text is included for images so users can understand the content even if they cannot see the images.

Developer Tools also help test and check the responsiveness of the website across different screen sizes.


What tools or resources did you find most helpful during this project?

I used the Per Scholas resources and the materials shared on Slack during the project. I also referred to W3Schools to learn more about icons and HTML styling. I used VS Code as my code editor to write the code, and I used the browser’s Developer Tools to debug and better understand where my code was working well and where improvements were needed.
